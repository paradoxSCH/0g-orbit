# User Prep Checklist

These are the concrete materials you should prepare while implementation starts. The project can be mocked without them, but final submission will be stronger if these are ready early.

## Wallet / Network

- A deployer wallet for 0G deployment.
- A separate demo owner wallet if possible.
- A separate authorized agent wallet if possible.
- Enough 0G testnet/mainnet funds for deployment and demo transactions.
- Confirm which 0G network we will deploy to for final submission.

## Branding

- Final project name: `0G Orbit` or alternative.
- Short tagline.
- Logo preference or simple text mark acceptance.
- Preferred color/style direction for the dashboard.

## Demo Scenario

- Pick the primary persona:
  - DAO ops lead.
  - startup founder.
  - autonomous research team.
  - AI SaaS operator.
- Pick the demo operations:
  - approved contributor payout.
  - approved subscription renewal.
  - unknown recipient rejection.
  - over-budget rejection.
  - optional high-risk contract call rejection.

## Submission Assets

- X account that will post the required public project post.
- YouTube/Loom account for demo video.
- GitHub account / organization for the public repository.
- Any team member names, handles, and roles you want listed.

## Technical Choices to Confirm

- Preferred frontend stack: likely Next.js + wagmi + viem.
- Preferred contract framework: Foundry or Hardhat.
- Whether to prioritize 0G mainnet or testnet deployment.
- Whether to include 0G Compute in final or keep it as a mocked/stubbed integration with a clear roadmap.

## Strong Optional Material

- One real-ish DAO operations workflow you have experienced.
- A short threat model: what could go wrong if an agent had unlimited wallet access?
- Any existing 0G docs/examples you want us to follow.
